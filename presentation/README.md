# Riptide RIA deck toolchain

Everything needed to rebuild the master deck and the technical documentation from source.
These artifacts are code, not hand-edited files: change a line, rebuild, re-run the gates.

Verified: a clean checkout of this bundle rebuilds both artifacts and passes every gate.

## Setup

```bash
npm install                    # pptxgenjs, docx
pip install matplotlib pillow

# Install the three fonts in fonts/ (Inter, Inter-Italic, Source Serif 4).
# Windows: select all three, right-click, Install.
# The build references them by name. Without them, renders silently substitute and are wrong.
```

Paths are resolved relative to the scripts, so the bundle runs from anywhere. Outputs land
in `out/`.

## Build

```bash
npm run all          # diagrams, deck, doc, then the contrast gate

# or individually:
npm run diagrams     # regenerate both diagrams (overlap, containment, vendor-name gates)
npm run deck         # -> out/Riptide-RIA-Master-Deck.pptx   (38 slides)
npm run doc          # -> out/Riptide-RIA-Technical-Documentation.docx  (18 pages)
npm run gate         # contrast: "705 runs / 0 failures" + layout: "geometry OK, aspect OK, text fit OK"
```

## Layout

| Path | Role |
|---|---|
| `build/build_master_v6.js` | The deck. Design tokens, grid constants, and helpers (`slide`, `panel`, `code`, `grid`, `mark`) at the top; then 38 slides in order. |
| `build/build_tech_doc_v2.js` | The document. Same token block, then sections 1-9. |
| `diagrams/fix_diagrams.py` | Both diagrams. Carries three gates: box overlap, label containment, and the vendor-name scan on the logical layer. |
| `qa/contrast_gate.py` | Background-aware WCAG AA gate. Resolves each text run's real backdrop (enclosing filled panel, else slide canvas) before measuring. Usage: `python3 qa/contrast_gate.py <file.pptx>` |
| `qa/layout_gate.py` | Geometry, image aspect ratio, and text fit. Re-measures every run with real font metrics. Usage: `python3 qa/layout_gate.py <file.pptx>`. Exit 0 clean, 1 on failure, so CI can call it. |
| `assets/` | Both diagrams and both contour backgrounds. Regenerated by `npm run diagrams`. |
| `fonts/` | Inter, Inter Italic, Source Serif 4 (OFL). |
| `legacy/` | Superseded builders. Reference only; do not build from these. |

## Editing rules that keep it correct

**Colour.** Pure `TEAL` (#00C9B1) and `AMBER` (#F59E0B) for fills, bars, and text on dark.
`DTEAL` (#007A6A) and `DAMBER` (#9C5D00) for small text on light: the pure versions measure
1.97:1 and 2.01:1 on Bone and fail WCAG AA. Amber means human gate or exception. Nothing else.

**Images.** Always place at true aspect ratio. Both diagrams are exactly 16:9 (2880x1620), so
a width of 8.4 in requires a height of 4.725 in. Getting this wrong is what stretches fonts.

**The logical layer is vendor-free.** Slides 19-22 and `ria_logical.png` must contain no
product, vendor, or model name. `fix_diagrams.py` enforces this on the diagram; extend the
check if you add a logical slide. Slide 23 (instantiation mapping) is the deliberate exception.

**No em dashes.** Anywhere. Hyphens only for hyphenated words.

**Re-run the gates after any edit.** They exist because each has caught a real defect: a 27%
image stretch, 49 contrast failures, a source line colliding with a panel, a label overflowing
its box by 0.03 in, a duplicate table number.

## Caveat

Every artifact here was verified programmatically: geometry, image aspect, text fit with real
font metrics, WCAG contrast, and vendor-name scans. The rendering preview was unavailable
during the build session, so nothing has been checked by eye. The measurements are sound;
taste is not something they can check.
